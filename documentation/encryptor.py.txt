To use encryptor.py, the command line console or bash or shell must be opened in the same directory as it is located in. Encrpytor.py is coded to operate on folders and files within the same directory as itself. If you choose to use it on folders and files in other directories, you do so at your own risk, but the pathway is standard. If you don't do everything inside the same folder directory, add /path/to/ before your folder or filename.

Spaces and special characters (like &, #, ?, etc.) are allowed in file and folder names when enclosed in quotes. For example, "folder name!" and "file name.txt" are both valid.

Usage: python encryptor.py -encrypt/-decrypt/-check <path_to_directory> [metadata_file] [key]

Functions of encryptor.py:

    -encrypt: To encrypt the specified file or directory.
    -decrypt: To decrypt the specified file or directory.
    -check: To check the encryption status of files, or of files in a specified directory.

Examples of encryptor.py usage:

To encrypt, decrypt, and check the status of encryption on a folder with encryptor.py:
	python encryptor.py -encrypt "folder 1" metadata.json --encryptednamedfolder
	python encryptor.py -decrypt "encryptednamedfolder" metadata.json --"folder 1" 
	python encryptor.py -check "folder 1"  metadata.json  --not encrypted

To encrypt a file with encryptor.py:
	python encryptor.py -encrypt test.txt metadata.json --encryptednamedfile.enc 
	python encryptor.py -decrypt encryptednamedfile.enc metadata.json --test.txt 
	python encryptor.py -check encryptednamedfile.enc metadata.json --encrypted
